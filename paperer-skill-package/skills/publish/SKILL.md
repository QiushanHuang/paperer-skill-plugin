# Publish

Generate a **one-page, high-density HTML brief** from a prepared Markdown file. The output is a compact dashboard — not a linear article. The agent must curate and compress, not merely convert.

## Input

The user provides a path to a Markdown file. The Markdown was extracted from an academic paper and may contain 8+ sections with embedded figures, tables, and formulas.

Read the file using the Read tool. Note the directory path — the output HTML will be written to the same directory.

## Output

Write a single self-contained HTML file to the same directory as the input: `<input-name>-report.html`.

## Core Principle: Curate, Don't Convert

The input Markdown may be 3000+ words with dozens of sub-sections. Your job is NOT to convert it all. Your job is to **judge what matters** and present only that, at maximum density.

Target: **one viewport on a 1080p screen** (~900px visible height). The reader should grasp the paper's essence without scrolling, or with minimal scrolling.

## Language

The HTML report language must match the input `summary.md` language. If the summary is in Chinese, all card labels, prose, and UI text must be in Chinese. If the summary is in English, use English throughout. The displayed page title always uses the Chinese translation regardless of body language.

## Content Curation Rules

Read the full Markdown, then select content for these cards. Each card has a **hard word budget**.

### Card 1: Header (always present)
- Paper title (Chinese translation as display title, English original smaller)
- Authors, venue, year
- One-sentence positioning from the source (taken from 论文定位 or generated)
- **No word budget — keep it naturally compact**

### Card 2: TL;DR (≤80 words)
- Sourced from 核心结论速览 (Section 1.3)
- Compress into: Problem → Method → Result → Contribution
- Each as ONE tight sentence, not a paragraph
- Use `→` or similar connectors for flow

### Card 3: Why It Matters (≤60 words)
- Sourced from 这篇论文为什么值得关注 (Section 1.2)
- What key question does it answer? What's the novelty against prior work?
- 2-3 sentences max

### Card 4: Method (≤100 words)
- Sourced from 整体方法框架 + 核心创新点 (Section 3.1, 3.2)
- Input → Core mechanism → Output, in plain language
- Highlight 1-2 key innovations with bold
- If there's a method architecture figure, embed it here (ONE figure max)

### Card 5: Key Results (≤80 words)
- Sourced from 关键结果 (Section 4.2)
- Extract the 2-4 most impressive quantitative results
- Present as a mini-table or bold metric highlights
- If there's a key results table/figure, embed it (ONE visual max)

### Card 6: Key Visuals (0-3 items)
- Select the **most informative** figures/tables from the full Markdown
- Skip redundant, decorative, or low-information visuals
- Each gets: image + one-line caption (≤15 words)
- If Method or Results already embedded a visual, don't repeat it here
- This card is OPTIONAL — skip if no high-value visuals remain

### Card 7: Key Equations (optional, ≤2 equations)
- Only include if the paper is math-heavy and equations are central to understanding
- Render each equation in LaTeX inside an `.equation-row` with a short `.eq-note` label
- Each `.equation-row` must have a `data-detail` attribute with a detailed explanation (2–6 sentences, ≤120 words) sourced from Section 5.3 of `summary.md` — this powers the click-to-expand lightbox
- Skip this card entirely for empirical/system papers

### Card 8: Limitations & Open Questions (≤60 words)
- Sourced from 论文的不足 + 可以进一步追问的问题 (Section 7)
- 2-3 bullet points of genuine weaknesses or open directions
- No filler — every point must be substantive

### Card 9: Paper Ratings (auto-generated)
- Sourced from `report.json` `ratings` field (generated by `literature-summary`)
- Display five dimensions as a horizontal bar or radar-style compact visual
- Each dimension shows: Chinese label, score (1–5 as filled/empty dots or stars), and one-sentence justification
- Dimensions: 创新性, 严谨性, 实用性, 清晰度, 影响力
- If `ratings` is absent from `report.json`, skip this card

### Card 10: Bottom Line (≤30 words)
- Sourced from 一句话总结 (Section 8.2)
- One precise, defensible closing sentence

## What to Drop

The following content from the input Markdown should generally be **excluded** from the brief:

- Extended background / related work narratives
- Per-figure, per-table, per-formula detailed walkthroughs (Section 5 bulk)
- Full experimental setup details (datasets, hyperparams, baselines enumeration)
- Repeated information across sections
- Author-acknowledged limitations that are obvious or generic
- Meta-commentary about extraction quality

## Term Glossary

The brief targets readers who may not be domain experts. When the curated text contains key technical terms that are essential to understanding but not self-explanatory, mark them as interactive glossary terms.

### How to mark terms

Wrap the term in a `<span class="term">` with a `data-def` attribute containing a one-sentence plain-language explanation (≤30 words):

```html
<span class="term" data-def="一种用于深度学习的优化算法，是 Adam 的改进版本，加入了解耦的权重衰减正则化">AdamW</span>
```

### What to mark

- Algorithm / method names that the paper's argument depends on (e.g., Newton-Schulz, ZeRO-1, MoE)
- Metric names that a non-specialist might not recognize (e.g., MMLU, Pareto frontier)
- Architecture-specific jargon (e.g., attention head, RMSNorm, bf16)

### What NOT to mark

- Terms already explained in the surrounding sentence
- Common abbreviations (LLM, GPU, API)
- Terms that appear in card labels or headings
- More than ~8–12 terms per report — over-marking creates noise

### Interaction

On click, a floating card appears below the term showing the definition. Clicking again or clicking elsewhere dismisses it. The CSS and JS for this are included in the template below.

## Visual Lightbox

Figures, tables, and formula images in the brief are compact by design. When a reader needs to inspect details, clicking the image opens a **full-viewport lightbox overlay** with the image at full resolution and a longer explanation.

### How to mark visuals

Add a `data-detail` attribute to each `<figure>` element. The value is a paragraph-length explanation (2–4 sentences, ≤80 words) sourced from the full `summary.md` Section 5 (图表与公式解读). This provides richer context than the inline `<figcaption>`.

```html
<figure data-detail="该图展示了 Muon 和 AdamW 在不同模型规模下的 scaling law 曲线。蓝色实线（Muon）在所有规模下系统性地低于红色虚线（AdamW），表明 Muon 在相同计算预算下始终获得更低的训练损失。">
  <img src="assets/figures/fig-003.png" alt="...">
  <figcaption>短标题</figcaption>
</figure>
```

Formula `.equation-row` elements also support lightbox. Add `data-detail` to the `.equation-row` element with a detailed derivation or explanation (2–6 sentences, ≤120 words). The lightbox re-renders the equation at a larger size with the full explanation below.

```html
<div class="equation-row" data-detail="该公式定义了 Muon 优化器的核心更新规则。首先计算梯度动量 M_t，然后通过 Newton-Schulz 迭代对其正交化得到 O_t，最后执行权重更新。正交化步骤确保更新方向在谱范数意义下被标准化，这是 Muon 相对于 AdamW 的核心差异。">
  <div class="math-display">$$\mathbf{W}_t = \mathbf{W}_{t-1} - \eta_t \mathbf{O}_t$$</div>
  <div class="eq-note">Eq. 1: Muon 核心更新规则</div>
</div>
```

### What to include in `data-detail`

For figures:
- What the visual shows (structure, axes, comparison)
- Key observation or takeaway
- Role in the paper's argument

For equations:
- What the formula computes and why
- Key variable meanings (if not obvious from the inline `eq-note`)
- Derivation context or connection to other equations
- Role in the paper's method or proof

### Interaction

**Figures**: Click image → lightbox overlay (dark backdrop, centered image at `max-width: 90vw; max-height: 70vh`, explanation text below). Click backdrop or press Escape → dismiss.

**Equations**: Click anywhere on the `.equation-row` → lightbox overlay with the equation re-rendered at larger scale (`font-size: 1.4em`) and the full `data-detail` explanation below. Click backdrop or press Escape → dismiss.

The CSS and JS are self-contained in the template below.

## Layout Specification

The layout is a **responsive card grid**, not a linear article.

### Desktop (≥900px): Two-column grid
```
┌─────────────────────────────────────────┐
│  Header: Title + Authors + Position     │
├────────────────────┬────────────────────┤
│  TL;DR             │  Why It Matters    │
├────────────────────┴────────────────────┤
│  Method (full width, may include fig)   │
├────────────────────┬────────────────────┤
│  Key Results       │  Limitations       │
├────────────────────┴────────────────────┤
│  Key Visuals (horizontal scroll/grid)   │
├─────────────────────────────────────────┤
│  Bottom Line                            │
└─────────────────────────────────────────┘
```

### Mobile (<900px): Single column stack

All cards stack vertically. Key Visuals becomes a vertical list.

## Instructions

1. Read the Markdown file
2. Determine the output path: same directory, `<basename>-report.html`
3. **Analyze the content** — identify the most important information for each card
4. **Curate ruthlessly** — apply the word budgets, drop low-value content
5. Generate the HTML following the template and CSS below
6. Write the HTML file
7. Tell the user the output path

      var top = tr.bottom + 8;

      if (left < 8) left = 8;
      if (left + tipW > window.innerWidth - 8) left = window.innerWidth - tipW - 8;

      var arrowLeft = Math.max(16, Math.min(tipW - 16, tr.left + tr.width / 2 - left));
      tip.style.left = left + 'px';
      tip.style.top = top + 'px';
      tip.style.setProperty('--arrow-left', arrowLeft + 'px');

      activeTip = tip;
    });
  })();

  // --- Visual & equation lightbox ---
  (function() {
    function closeLightbox() {
      var lb = document.querySelector('.lightbox');
      if (lb) lb.remove();
    }

    function openImageLightbox(imgSrc, caption, detail) {
      var lb = document.createElement('div');
      lb.className = 'lightbox';
      lb.innerHTML =
        '<button class="lightbox-close">&times;</button>' +
        '<img src="' + imgSrc + '" alt="">' +
        (caption ? '<div class="lightbox-caption">' + caption + '</div>' : '') +
        (detail ? '<div class="lightbox-detail">' + detail + '</div>' : '');
      lb.addEventListener('click', function(e) {
        if (e.target === lb || e.target.classList.contains('lightbox-close')) lb.remove();
      });
      document.body.appendChild(lb);
    }

    function openEquationLightbox(mathHtml, label, detail) {
      var lb = document.createElement('div');
      lb.className = 'lightbox';
      var eqDiv = document.createElement('div');
      eqDiv.className = 'lightbox-equation';
      eqDiv.innerHTML =
        (label ? '<div class="eq-label">' + label + '</div>' : '') +
        '<div class="math-display">' + mathHtml + '</div>';
      lb.innerHTML = '<button class="lightbox-close">&times;</button>';
      lb.appendChild(eqDiv);
      if (detail) {
        var detailDiv = document.createElement('div');
        detailDiv.className = 'lightbox-detail';
        detailDiv.textContent = detail;
        lb.appendChild(detailDiv);
      }
      lb.addEventListener('click', function(e) {
        if (e.target === lb || e.target.classList.contains('lightbox-close')) lb.remove();
      });
      document.body.appendChild(lb);
      // Re-render KaTeX inside lightbox
      if (typeof renderMathInElement === 'function') {
        renderMathInElement(eqDiv, {
          delimiters: [
            {left: '$$', right: '$$', display: true},
            {left: '$', right: '$', display: false}
          ]
        });
      }
    }

    document.addEventListener('keydown', function(e) {
      if (e.key === 'Escape') closeLightbox();
    });

    document.addEventListener('click', function(e) {
      // Handle image clicks (figures, visuals)
      var img = e.target.closest('.card figure img, .visuals-grid img');
      if (img) {
        e.stopPropagation();
        var figure = img.closest('figure');
        var caption = figure?.querySelector('figcaption')?.textContent || '';
        var detail = figure?.dataset.detail || '';
        openImageLightbox(img.src, caption, detail);
        return;
      }

      // Handle equation-row clicks
      var eqRow = e.target.closest('.equation-row');
      if (eqRow) {
        e.stopPropagation();
        var mathEl = eqRow.querySelector('.math-display');
        var mathHtml = mathEl ? mathEl.innerHTML : '';
        var label = eqRow.querySelector('.eq-note')?.textContent || '';
        var detail = eqRow.dataset.detail || '';
        openEquationLightbox(mathHtml, label, detail);
        return;
      }
    });
  })();
  </script>
</body>
</html>
